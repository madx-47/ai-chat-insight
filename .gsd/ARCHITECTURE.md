## Data Flow

1. **User initiates analysis** (CLI picks file OR web uploads JSONL)
2. **Pipeline reads JSONL** → parses into ChatRecord array (`reader.js`)
3. **Records grouped by session** → Map<sessionId, records[]> (`reader.js`)
4. **Metrics computed** → quantitative data (`metrics.js`)
5. **Conversational sessions filtered** → only user+assistant exchanges (`reader.js`)
6. **LLM analyzes each session** → SessionFacets per session (`providers/*.js`)
7. **Facets aggregated** → cross-session statistics (`aggregator.js`)
8. **Qualitative insights generated** → narrative insights (`qualitative/*.js`)
9. **Insight JSON written** → `output/insight-<timestamp>.json`
10. **HTML report generated** → `output/reports/insight-<timestamp>.html`
11. **Dashboard displays** (web) or CLI shows summary
